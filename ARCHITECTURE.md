# Architecture

本项目当前处于可运行原型阶段，架构重点是：**自然语言 → 结构化计划 → 安全执行 → 确认闭环**。

## 目标
- 支持自然语言驱动的 Windows 操作（PowerShell 为主）
- 保留可扩展的能力接口（LLM / 工具 / 记忆）
- Host 只负责 UI，核心逻辑集中在 Kernel 扩展与服务层

## 组件划分（当前实现）

### 1. Abstractions
定义任务与能力的基础契约（任务、能力、执行上下文等）。

### 2. Core
任务执行引擎与能力注册中心，负责调度能力执行。

### 3. Extensions
提供具体能力实现，例如：
- Chat：聊天回复能力
- Shell：PowerShell 任务规划与执行能力

### 4. Providers
外部服务接入（当前为智谱 LLM）。

### 5. Host (WPF)
UI 与交互层，只负责展示和输入。

## 核心流程（当前实现）
1. 用户输入自然语言
2. LLM 规划结构化任务（意图、目标、风险级别）
3. Kernel 进行本地验证与安全策略处理
4. 高风险操作进入确认
5. 执行 PowerShell 命令并回显结果

## 安全策略
高风险操作（删除、移动、覆盖等）必须进入确认流程，避免误操作。

## 扩展方向（下一阶段）
- 上下文管理与多轮任务编排
- 多模态输入（图像/语音/屏幕）
- 记忆与检索（RAG / 向量库）
- 多 Agent 与权限隔离
